# 네이버 오픈API 공통 가이드 요약

네이버 오픈API를 사용하여 애플리케이션을 개발할 때 필요한 공통적인 사전 준비 사항, 인증 방식 및 호출 규격을 정리한 가이드입니다.

---

## 1. 개요 및 인증 방식
네이버 오픈API는 크게 **로그인 방식**과 **비로그인 방식**으로 나뉩니다.
본 문서에서 다루는 검색 및 데이터랩 API는 모두 **비로그인 방식 오픈 API**에 해당합니다.

* **비로그인 방식 오픈 API**: 사용자 로그인 없이, 애플리케이션 등록 시 발급받은 **Client ID**와 **Client Secret** 헤더 값만 HTTP 요청에 포함하여 호출하는 방식입니다.

## 2. 사전 준비 사항
API를 호출하기 위해서는 [네이버 개발자 센터](https://developers.naver.com/)에 애플리케이션을 등록하고 인증키를 발급받아야 합니다.

1. **애플리케이션 등록**: `Application > 애플리케이션 등록` 메뉴 진입
2. **API 설정**: 사용하려는 API(예: 검색, 데이터랩 등)를 선택 및 추가
3. **환경 설정**: 웹, Android, iOS 등 서비스 환경에 맞는 정보 입력
4. **인증키 확인**: 등록 완료 후 `내 애플리케이션 > 개요`에서 `Client ID` 및 `Client Secret` 확인

## 3. 공통 HTTP 요청 헤더
비로그인 방식 API 호출 시 HTTP 요청 헤더에 다음 두 필드를 반드시 포함해야 합니다.

| 헤더 이름 | 설명 |
| :--- | :--- |
| `X-Naver-Client-Id` | 발급받은 애플리케이션의 Client ID 값 |
| `X-Naver-Client-Secret` | 발급받은 애플리케이션의 Client Secret 값 |
| `Content-Type` | `application/json` (POST 요청 시) 또는 필요에 따라 인코딩 설정 |

## 4. 공통 오류 코드 개요
API 호출이 실패할 경우 HTTP 상태 코드와 함께 오류 정보가 반환됩니다.

* **400 (Bad Request)**: 잘못된 요청 (파라미터 누락, 부적절한 값, 인코딩 오류 등)
* **403 (Forbidden)**: 권한 없음 (내 애플리케이션 설정에서 해당 API를 활성화했는지 확인 필요)
* **404 (Not Found)**: 존재하지 않는 API 주소 확인
* **500 (Internal Server Error)**: 네이버 서버 내부 오류 (개발자 포럼에 문의 필요)
