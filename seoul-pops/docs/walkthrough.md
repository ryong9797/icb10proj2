# 서울 생활인구 데이터 EDA 대시보드 구현 완료

요청하신 `seoul-pops/data/LOCAL_PEOPLE_DONG_202606.parquet` 데이터를 기반으로 하는 Streamlit 탐색적 데이터 분석(EDA) 대시보드 구축이 성공적으로 완료되었습니다.

대시보드는 동적 컬럼 스캔 방식을 채택하여 사전에 컬럼명을 정확히 알지 못하더라도 자동으로 수치형 및 범주형 변수를 식별하고 최적의 분석 결과를 제공합니다.

## 구현된 주요 기능

1. **데이터 로딩 및 전처리 최적화 (`utils.py`)**
   - `@st.cache_data`를 이용해 100MB 이상의 대규모 Parquet 파일을 메모리에 한 번만 로드하여 속도를 향상시켰습니다.
   - Parquet 데이터와 Excel 형태의 행정동 매핑 데이터를 조인하여 분석 가능성을 높였습니다.

2. **Overview (데이터 개요)**
   - 데이터프레임 미리보기, 행/열 수 표시
   - 각 컬럼별 데이터 타입 확인
   - 결측치 탐지 및 요약 통계량 (수치형) 계산

3. **Item Analysis (단일 항목 분포 파악)**
   - 선택된 변수의 상세 기술 통계 제공 (최소/최대/평균/중앙값/왜도/첨도 등)
   - [Plotly](https://plotly.com/python/)를 이용한 히스토그램 및 박스플롯(Boxplot)으로 분포 확인 및 이상치(Outlier) 식별

4. **Correlation (상관성 분석)**
   - 다중공선성 파악을 위한 피어슨 상관계수 행렬(Correlation Matrix Heatmap) 시각화
   - 원하는 두 수치형 변수 간의 관계를 확인할 수 있는 동적 산점도(Scatter Plot) 시각화

## 🚀 실행 방법 (테스트)

터미널에서 아래의 명령어를 입력하여 대시보드를 직접 실행해 보실 수 있습니다.

```bash
uv run streamlit run seoul-pops/src/app.py
```

명령어를 실행하시면 브라우저에서 자동으로 대시보드가 열리며, 좌측 사이드바를 통해 `Overview`, `Item Analysis`, `Correlation` 탭을 이동하며 데이터를 탐색할 수 있습니다.

> [!TIP]
> **성능 최적화 팁**
> 대시보드 내 시각화 속도 개선을 위해 산점도나 히스토그램의 경우 기본적으로 1만 개~10만 개의 무작위 표본(Sampling) 데이터만 그리도록 처리해두었습니다. 전체 데이터를 그리려면 코드 내 `.sample(...)` 로직을 조절하시면 됩니다.
